# T-Invest Local Dashboard MVP

Маленький локальный dashboard для просмотра портфеля и облигаций из T-Invest API на вашем компьютере. Проект ориентирован на реальный API, без базы данных, Docker и авторизации пользователей внутри приложения.

## Что внутри

- `server/` - Express + TypeScript backend.
- `server/src/services/tinvestClient.ts` - тонкий клиент официального T-Invest REST API.
- `server/src/services/portfolioService.ts` - сбор портфеля, облигаций, цен, купонов и summary.
- `client/` - Vite + React + TypeScript frontend.
- `client/src/App.tsx` - одна страница dashboard со всеми счетами, графикой распределения, таблицей всех позиций и деталями по облигациям.

Для локального MVP используется официальный REST-прокси T-Invest API:

`https://invest-public-api.tbank.ru/rest`

Это тот же официальный API T-Invest, но без необходимости подключать gRPC SDK и нативные зависимости.

## Требования

- Node.js 16.16 или новее. Рекомендуется Node.js 20 LTS.
- npm
- Read-only токен T-Invest API

## Установка

```bash
npm install
```

Для Codex Cloud environment используйте тот же setup command:

```bash
npm install
```

Проверочный command:

```bash
npm run typecheck && npm run build
```

Создайте локальный `.env` из примера:

```bash
cp .env.example .env
```

Откройте `.env` и вставьте токен:

```env
TINVEST_TOKEN=ваш_read_only_токен
PORT=3001
TINVEST_BASE_URL=https://invest-public-api.tbank.ru/rest
TINVEST_ALLOW_INSECURE_TLS=false
TINVEST_REQUEST_TIMEOUT_MS=15000
```

Токен используется только backend-частью и не попадает во frontend.

## Запуск

```bash
npm run dev
```

Откройте приложение в браузере:

```text
http://localhost:5173
```

Backend будет доступен на:

```text
http://localhost:3001
```

Если `3001` уже занят, поменяйте `PORT` в `.env`, например на `3002`. Vite читает тот же `.env` и автоматически направит `/api/*` на выбранный backend-порт.

Если в локальной сети или прокси возникает ошибка `self signed certificate in certificate chain`, для dev-просмотра можно временно поставить `TINVEST_ALLOW_INSECURE_TLS=true`. Для обычного использования оставляйте `false`.

## REST endpoints

- `GET /api/health`
- `GET /api/accounts`
- `GET /api/portfolio?accountId=...`
- `GET /api/positions?accountId=...`
- `GET /api/bonds?accountId=...`
- `GET /api/summary?accountId=...`
- `GET /api/dashboard`

1. Центральный обзор по всем счетам и переключение на отдельный счет.
2. Сводные карточки стоимости, позиций, активных счетов и текущего результата.

6. Панель лидеров и просадок по доходности.
7. Валютная экспозиция по native currency позиций.
8. Risk/quality flags: концентрация, просадки, нулевая оценка, blocked-позиции, foreign currency.
9. Поиск, фильтры, сортировка таблицы и CSV export текущего списка.

11. Компактный monitor mode для отдельного экрана: стоимость, результат, счета, аллокация, топ позиций, купоны и контрольные сигналы на первом экране.
12. Темная тема по умолчанию и переключение на светлую тему.
13. Режим "Экран" для постоянного монитора: скрывает рабочие детали и оставляет главный dashboard.
14. Автопереключение счетов каждые 30 секунд.
15. Countdown до следующего polling-обновления.
16. Общий календарь ближайших событий: купоны и погашения.
17. Купонный доход за ближайшие 30 дней.
18. Месячные бары будущих купонов на 12 месяцев.
19. Панель качества данных: отсутствующие цены, нулевые оценки, blocked и облигации без купонов.
20. Топ облигаций по ожидаемым купонам на 12 месяцев.
21. Верхняя строка инсайтов: крупнейшая позиция, лучший/худший инструмент, ближайшее событие и статус данных.
22. Heatmap позиций: размер плитки показывает долю в портфеле, цвет показывает плюс или минус.
23. Панель концентрации: крупнейшая позиция, доля топ-5 и доля валютной части.
24. Live-тренд сессии: изменение стоимости с момента открытия и с последнего polling-тика.
25. Мини-график стоимости без базы данных, на основе последних frontend snapshots.
26. Fullscreen-кнопка и горячая клавиша для отдельного монитора.
27. Wake Lock-кнопка, чтобы браузер по возможности не давал экрану уснуть.
28. Доходность по типам активов: где результат создается или теряется.
29. Health score портфеля от 0 до 100 на основе концентрации, просадок, качества данных и blocked-позиций.
30. Очередь внимания: короткий список того, куда стоит посмотреть первым.
31. Countdown до следующего автопереключения счета и название следующего среза.
32. Session high/low и просадка от максимума текущей frontend-сессии.
33. Кнопка `Сброс live` для очистки локальной frontend-истории.
34. Измерение latency backend/API-запроса на каждом обновлении.
35. Индикатор свежести данных: `Live`, `Stale`, `API error`.
36. Панель свежести данных с возрастом последнего успешного ответа.
37. Рейтинг счетов по текущему результату.
38. Кнопка `Monitor link` для копирования ссылки на экранный режим.
39. Метрики диверсификации: HHI, эффективное число позиций, доля топ-10 и число малых позиций.
40. Купонный cashflow forecast на 30/90/365 дней и среднемесячный прогноз.
41. Структурные подсказки по концентрации, топ-5, валютной подушке и облигационному cashflow.
42. Backend timeout для запросов к T-Invest API, чтобы монитор не подвисал на сетевых сбоях.
43. Один мягкий retry для временных ошибок `408/429/5xx`.
44. Локальные часы и uptime frontend-сессии на monitor screen.
45. Короткий pulse-текст: здоровье, результат, cashflow и ближайшее событие одной строкой.

Горячие клавиши:

- `K` - включить или выключить режим "Экран".
- `A` - включить или выключить автопереключение счетов.
- `F` - включить или выключить fullscreen.
- `W` - включить или выключить удержание экрана через Wake Lock API, если браузер поддерживает.
- `Esc` - выйти из режима "Экран".

Также можно открыть monitor-view сразу:

```text
http://localhost:5173?mode=monitor&auto=1
```

## Ограничения MVP

- Нет базы данных.
- Нет истории портфеля.
- Есть только frontend-история текущей сессии в localStorage; полноценная история требует базы.
- Нет стриминга.
- Данные обновляются polling-ом раз в 10 секунд.
- Нет авторизации пользователей в локальном приложении.
- Расчет доходности по облигациям сделан best-effort на основе данных портфеля, НКД, номинала, текущих цен и `InstrumentsService/GetBondCoupons`.
- Купоны и справочники инструментов кэшируются в памяти backend-процесса, чтобы polling раз в 10 секунд не дергал справочные endpoints без необходимости.
- Временные ошибки T-Invest API повторяются один раз; долгие запросы ограничены `TINVEST_REQUEST_TIMEOUT_MS`.
- Общая таблица показывает все позиции, которые возвращает portfolio endpoint: акции, облигации, фонды, валюты и другие типы инструментов.
